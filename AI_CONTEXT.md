# AI Context: calls-dashboard

Этот файл — короткая оперативная документация для любого AI-агента, который продолжит работу в проекте.

## 1) Текущая цель проекта

Проект содержит два независимых статических дашборда и главное меню:

- `sravnenie-calls/` — старый дашборд (сравнение февр-мар 2026).
- `qual-dashboard/` — новый дашборд по отчету "Квал лиды дек25-мар26".
- `index.html` в корне — главное меню со ссылками на оба раздела.

Важно: дашборды должны оставаться независимыми. Общего runtime-кода между ними нет.

## 2) Структура (фактическая)

- `index.html` — главное меню.
- `sravnenie-calls/`
  - `index.html`, `days.html`, `weeks.html`, `urman-vr-mar.html`
  - `css/common.css`
  - `js/source-store.js`, `js/data-loader.js`, `js/month-aggregate.js`
  - `data/catalog.json`, `data/vk.json`, `data/bundled.js`
- `qual-dashboard/`
  - `index.html` (Обзор)
  - `sources.html` (Сравнение по источникам)
  - `roles.html` (Сравнение по ролям)
  - `css/common.css`
  - `js/qual-data-loader.js`, `js/qual-aggregates.js`, `js/qual-ui.js`
  - `data/qual-leads-dec25-mar26.csv`

## 3) Что уже реализовано в новом дашборде

- Сравнение всегда по 4 периодам: `дек.25`, `янв.26`, `февр.26`, `мар.26`.
- Метрики в сравнении:
  - Лиды
  - Квал
  - CR квал
  - Презентации
  - CR презентации
  - Скайп
  - CR скайп
  - Дозвоны
  - CR дозвон
- График "Лиды" на страницах сделан главным (на всю ширину сверху), остальные графики идут попарно.
- Data labels на графиках включены.

## 4) Специальная логика страницы "Сравнение по источникам"

Файл: `qual-dashboard/sources.html`

- Есть фильтр `Источник`.
- По умолчанию выбирается первый источник из списка.
- Опция "Все источники" для этого фильтра отключена.
- Круговые диаграммы долей удалены (по запросу), оставлен полный набор сравнительных графиков.

## 5) Локализация (RU)

- Навигационный пункт называется `Обзор` (не `Overview`).
- Интерфейс и подписи в новом разделе на русском.

## 6) Правила, которые нельзя ломать

1. Не связывать `qual-dashboard` с кодом из `sravnenie-calls`.
2. Не переносить root entrypoint с `index.html` (это главное меню для GitHub Pages).
3. При обновлении данных нового раздела менять только:
   - `qual-dashboard/data/qual-leads-dec25-mar26.csv`
4. Не переименовывать `sravnenie-calls/` обратно.

## 7) Как обновить данные (новый дашборд)

1. Выгрузить Google Sheet в CSV.
2. Заменить `qual-dashboard/data/qual-leads-dec25-mar26.csv`.
3. Проверить страницы:
   - `qual-dashboard/index.html`
   - `qual-dashboard/sources.html`
   - `qual-dashboard/roles.html`

## 8) Git/деплой

- Репозиторий: `https://github.com/GialchF5/calls-dash.git`
- Основная ветка: `main`
- GitHub Pages должен открывать корневой `index.html`.

## 9) Быстрый чек после изменений

- Открывается `index.html` (меню).
- Ссылка "старый дашборд" ведет в `sravnenie-calls/index.html`.
- Ссылка "новый дашборд" ведет в `qual-dashboard/index.html`.
- В `sources.html` фильтр источника по умолчанию выбран первым значением.
- Нет ошибок в консоли браузера при загрузке графиков.
